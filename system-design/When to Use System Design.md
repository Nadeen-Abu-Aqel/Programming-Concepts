# 🧠 When to Use System Design

## 📌 التعريف
System Design هو عملية تخطيط وبناء هيكل النظام قبل كتابة الكود، بهدف إنشاء نظام قابل للتوسع، سريع، ومستقر.

---

## 💡 الفكرة الأساسية
System Design لا يُستخدم داخل الكود، بل قبل كتابة الكود.

👉 هو مرحلة تفكير وتخطيط:
- كيف سيتم بناء النظام؟
- كيف سيتم التعامل مع المستخدمين؟
- كيف سيتم تخزين البيانات؟

---

## 🔄 كيف يعمل؟
- تحديد مكونات النظام (Servers, Database, APIs)  
- اختيار طريقة الاتصال بين الأجزاء  
- تحديد كيف يتم توزيع الضغط  
- التخطيط للأداء والتوسع  

---

## 🎯 متى نستخدمه؟

### 🔹 1. عند بناء أنظمة كبيرة
- عدد مستخدمين كبير  
- بيانات كثيرة  
- حاجة لأداء عالي  

---

### 🔹 2. قبل تطوير Backend معقد
- تطبيقات Web  
- أنظمة تحتوي على APIs  
- أنظمة متعددة الخدمات  

---

### 🔹 3. عند وجود مشاكل أداء
- بطء في النظام  
- ضغط عالي على السيرفر  
- انهيار النظام  

👉 نستخدم:
- Caching  
- Load Balancing  
- Scaling  

---

### 🔹 4. في المقابلات التقنية
- أسئلة تصميم الأنظمة  
- تقييم طريقة التفكير وليس الكود  

---

## ⚠️ ملاحظات
- لا يُستخدم في المشاريع الصغيرة  
- يزيد التعقيد إذا استُخدم بدون حاجة  
- يعتمد على فهم المشكلة أولًا  

---

## 🔥 مثال بسيط

❌ بدون System Design:  
سيرفر واحد + قاعدة بيانات واحدة  
→ ينهار عند زيادة المستخدمين  

---

✅ مع System Design:  
توزيع النظام على عدة سيرفرات  
→ أداء أفضل واستقرار أعلى  

---

## 📚 الخلاصة
System Design = تخطيط النظام قبل التنفيذ  

👉 يُستخدم عندما:
- يكبر النظام  
- تحتاج أداء عالي  
- تريد نظام قابل للتوسع 👍
