# 📚 Library

## 📌 التعريف
الـ Library (مكتبة) هي مجموعة من الدوال (Functions) أو الكلاسات الجاهزة التي يمكنك استخدامها لتنفيذ مهام معينة بدون كتابة الكود من الصفر.

## 💡 الفكرة الأساسية
بدل ما تبني البرنامج من البداية:
- Library تعطيك دوال جاهزة
- وانت تستدعيها وقت ما تحتاج

## 🔄 كيف يعمل؟
أنت تتحكم في تدفق البرنامج (Control Flow)
وتستدعي الدوال من الـ Library عند الحاجة

👉 هذا يعني: You call the Library

## 🔧 ماذا يوفر؟
كود جاهز (Reusable Code)  
تنفيذ مهام محددة  
توفير الوقت  
تقليل الأخطاء  
تسهيل التطوير  

## 🧠 أمثلة على Libraries

### Web (Frontend)
- jQuery  
- Axios  

### Backend / General
- Lodash (JavaScript)  
- Gson (Java)  

### Python
- NumPy  
- Pandas  

## ⚖️ الفرق بين Framework و Library

| Feature | Framework | Library |
|--------|----------|--------|
| التحكم | هو يتحكم بالكود | أنت تتحكم |
| الاستخدام | هيكل كامل | جزء صغير |
| مثال | Angular | jQuery |

## 🎯 متى نستخدم Library؟
- لما تحتاج وظيفة محددة  
- لما بدك تختصر كود معين  
- لما المشروع بسيط أو متوسط  
- لما بدك تحكم كامل بالكود  

## ⚠️ ملاحظات
- Library مش برنامج كامل  
- تستخدم داخل مشروعك  
- ممكن تستخدم أكثر من Library بنفس المشروع  

## 🔥 مثال بسيط

### بدون Library:
تكتب كل شيء بنفسك (مثلاً: request HTTP)

### مع Library:
تستخدم دوال جاهزة  
تختصر الكود  

## 📚 الخلاصة
Library هي أداة تساعدك تنجز مهام محددة بسرعة،  
لكن التحكم الكامل يظل بيدك 👍
