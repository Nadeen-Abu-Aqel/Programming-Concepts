📚 Library
📌 التعريف

الـ Library (مكتبة) هي مجموعة من الدوال (Functions) أو الكلاسات الجاهزة التي يمكنك استخدامها لتنفيذ مهام معينة بدون كتابة الكود من الصفر.

💡 الفكرة الأساسية

بدل ما تكتب كود من البداية:

تستخدم دوال جاهزة من الـ Library
وتستدعيها وقت ما تحتاج

👉 أنت المتحكم في الكود، والـ Library مجرد أداة تساعدك

🔄 كيف يعمل؟
أنت تكتب الكود وتتحكم في تدفق البرنامج (Control Flow)
وعند الحاجة تستدعي دوال من الـ Library

👉 هذا يعني:
You call the Library, not the opposite

🔧 ماذا يوفر؟
♻️ إعادة استخدام الكود (Reusable Code)
⏱️ توفير الوقت
🧩 تنفيذ مهام جاهزة (مثل: حسابات، API، UI)
🐞 تقليل الأخطاء
📦 تنظيم الكود
🧠 أمثلة على Libraries
🌐 Web
jQuery
Axios
🐍 Python
NumPy
Pandas
⚙️ Java
Apache Commons
Gson
⚖️ الفرق بين Library و Framework
Feature	Library	Framework
التحكم	أنت تتحكم بالكود	هو يتحكم بالكود
الاستخدام	جزء صغير	هيكل كامل
طريقة العمل	أنت تستدعيها	هو يستدعي كودك
مثال	jQuery	Angular
🎯 متى نستخدم Library؟
لما تحتاج وظيفة محددة (مثل إرسال طلب HTTP)
لما بدك تختصر وقت كتابة كود معين
لما يكون المشروع بسيط أو متوسط
لما بدك تحافظ على تحكم كامل بالكود
⚠️ ملاحظات
❌ Library مش برنامج كامل
✅ تستخدم داخل مشروعك
⚠️ ممكن تحتاج أكثر من Library في نفس المشروع
🔥 مثال بسيط
بدون Library:
تكتب كل الكود بنفسك (مثلاً: request HTTP)
مع Library:
تستخدم دالة جاهزة
تختصر الكود بشكل كبير
📚 الخلاصة

الـ Library هي أداة تساعدك على:

تسريع التطوير
تقليل الجهد
استخدام كود جاهز بدل إعادة كتابته

👉 الفرق الأساسي:
أنت تتحكم، وهي تساعدك فقط
